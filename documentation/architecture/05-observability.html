Chapter 05 · Platform

Observability

Two observability stacks, one OpenTelemetry pipeline. Whether you choose Azure Monitor or the Grafana OSS stack, the harness exports the same traces, metrics, and logs.

Two paths, same data

The harness uses OpenTelemetry (OTel) as its instrumentation standard. This means you choose where the telemetry goes — Azure-native or open-source — without changing any application code. The same spans, metrics, and log records flow regardless of the exporter.

Run both simultaneously

You can run both paths at the same time. The OTel SDK supports multiple exporters. Many teams send traces to Grafana/Tempo for developer debugging and metrics to Azure Monitor for operations/alerting.

What the harness instruments

The harness emits structured spans for every major subsystem. These spans carry semantic attributes that let you slice traces by model, tool, quality score, or token count.

Distributed traces (spans)

Span Source What It Tracks
agent.turn Agent runtime Full agent turn: prompt to response (includes tool calls)
agent.tool_call Tool executor Individual tool execution with args and result
rag.retrieve RAG pipeline Full retrieval: query transform, search, rerank, assemble
rag.embed Embedding service Embedding generation time and token count
mcp.request MCP server Incoming MCP tool/prompt/resource request
knowledge.query Knowledge graph Graph traversal query with node/edge counts
governance.drift Drift detector EWMA quality score, severity level
pipeline.behavior MediatR pipeline Each pipeline behavior execution time

Custom metrics (counters & histograms)

Metric Type What It Measures
harness.agent.turns Counter Total agent turns processed
harness.agent.tokens Histogram Token usage per turn (prompt + completion)
harness.rag.latency Histogram RAG retrieval latency in ms
harness.tools.invocations Counter Tool invocations by tool name
harness.governance.circuit_breaks Counter Circuit breaker trips by provider
harness.knowledge.graph_queries Histogram Graph query latency

Azure Monitor setup

For the Azure-native path, configure the OTel exporter to send data to Application Insights via its connection string.

json 
{
  "OpenTelemetry": {
    "Exporter": "AzureMonitor",
    "AzureMonitor": {
      "ConnectionString": "InstrumentationKey=xxx;IngestionEndpoint=https://eastus-8.in.applicationinsights.azure.com/"
    }
  }
}

Application Insights provides built-in dashboards for request rates, failure rates, and response times. For agent-specific views, create custom Azure Workbooks that query the customEvents and dependencies tables in Log Analytics.

KQL query example

Agent turn analysis — average and p95 duration by span name, bucketed by hour:

kql 
dependencies
| where name startswith "agent."
| summarize avg(duration), percentile(duration, 95), count()
  by name, bin(timestamp, 1h)
| render timechart

Grafana + Tempo + Prometheus setup

For the OSS path, point the OTel exporter at your OTel Collector endpoint.

json 
{
  "OpenTelemetry": {
    "Exporter": "Otlp",
    "Otlp": {
      "Endpoint": "http://otel-collector:4317",
      "Protocol": "grpc"
    }
  }
}

The OTel Collector receives spans, metrics, and logs, then routes them to Tempo (traces), Prometheus (metrics), and Loki (logs). Grafana provides the unified dashboard layer across all three backends. This stack runs as sidecar containers in the Container Apps environment or as a separate deployment.

✓
Pre-built Grafana dashboards

The harness ships with dashboard JSON files in tools/grafana/ that you can import directly. They cover agent performance, RAG pipeline health, and governance metrics.

Tracing an agent conversation

When you open a trace in your viewer (Jaeger, Tempo, or Application Insights Transaction search), here is what you see for a typical agent turn that involves RAG retrieval and tool use:

agent.turn [1,240ms]
Root span. Total duration of the conversation turn. Carries agent.model, agent.tokens.prompt, and agent.tokens.completion attributes.
rag.retrieve [380ms]
Child span. The RAG pipeline kicked in — query transform, search, rerank, assemble. Includes rag.chunks.count and rag.quality.score.
agent.tool_call [file_system] [95ms]
Child span. The agent called the file_system tool. Attributes include tool.name, tool.args, and tool.result.status.
knowledge.query [120ms]
Child span. Knowledge graph was consulted. Carries graph.nodes.count and graph.edges.count for the traversal.
mcp.request [ext-tool] [210ms]
Child span. An external MCP tool was invoked. Includes the MCP server name, tool name, and round-trip latency.

Each span carries attributes like agent.model, agent.tokens.prompt, agent.tokens.completion, rag.chunks.count, and rag.quality.score that let you diagnose exactly where time and tokens went.

!
Token tracking matters for cost

The harness.agent.tokens histogram, combined with Azure OpenAI pricing, gives you a direct cost-per-conversation metric. Set alerting thresholds based on your budget to catch runaway agent loops before they burn through your token allocation.

Alerting

Recommended alert rules for production deployments. These apply to both Azure Monitor (metric alerts) and Grafana (Prometheus alert rules).

Alert Condition Action
Agent error rate > 5% agent.turn spans with error status Page on-call
RAG latency > 3s p95 rag.retrieve duration Scale AI Search replicas
Circuit breaker open governance.circuit_breaks counter spike Check provider health
Token spend > daily budget harness.agent.tokens sum Throttle non-critical requests
